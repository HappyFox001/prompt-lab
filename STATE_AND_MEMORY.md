# 数值化状态与事件记忆使用指南

## 🎯 功能概述

本系统通过XML格式实现了两个高级功能：
1. **数值化状态系统** - AI可以根据对话自动更新状态值（如好感度、信任度等）
2. **事件记忆系统** - AI自动提取并记录重要对话事件，形成长期记忆

## 📋 使用步骤

### 1. 打开上下文对话框

点击右上角的 **"上下文"** 按钮，切换到 **"状态与记忆"** 标签页。

### 2. 配置数值化状态

#### 添加状态
1. 点击 **"添加状态"** 按钮
2. 填写信息：
   - **名称**：如"好感度"、"信任度"
   - **当前值**：初始数值
   - **最小值/最大值**：数值范围
   - **描述**：状态的用途说明
   - **颜色**：进度条显示颜色
3. 点击 **"保存"**

#### 示例配置
```
名称: 好感度
当前值: 50
最小值: 0
最大值: 100
描述: 角色对用户的情感程度
颜色: #10b981 (绿色)
```

### 3. 启用事件记忆

在"状态与记忆"标签页中，打开 **"事件记忆"** 开关。

启用后，AI会：
- 自动识别重要对话事件
- 记录事件描述和重要性等级
- 在后续对话中参考这些事件

## 🔧 技术原理

### XML格式通信

#### 1. 发送给LLM的上下文
```xml
<numeric_states>
  <state id="affection" name="好感度" value="75" min="0" max="100"
         description="角色对用户的情感程度" />
  <state id="trust" name="信任度" value="60" min="0" max="100" />
</numeric_states>

<memory_events>
  <event timestamp="2026-03-06T12:00:00Z" importance="8">
    用户分享了童年的重要经历
  </event>
  <event timestamp="2026-03-06T13:30:00Z" importance="7">
    建立了共同的兴趣话题
  </event>
</memory_events>
```

#### 2. LLM的输出格式
```xml
<response>
  <content>
    我理解你的感受，谢谢你愿意和我分享这些。
  </content>
  <state_updates>
    <update id="affection" delta="+5" />
    <update id="trust" delta="+3" />
  </state_updates>
  <event>
    <importance>8</importance>
    <description>用户表达了深层次的情感需求</description>
  </event>
</response>
```

### 自动处理流程

1. **发送阶段**
   - 系统自动将当前状态和历史事件添加到prompt
   - 告知AI输出格式要求

2. **接收阶段**
   - 解析AI响应中的XML标签
   - 提取 `<content>` 作为显示内容
   - 提取 `<state_updates>` 更新状态值
   - 提取 `<event>` 添加新事件记录

3. **显示阶段**
   - 用户只看到纯文本内容
   - 状态在后台自动更新
   - 事件自动记录到列表

## 📊 状态更新示例

### 对话场景
```
用户: 今天心情很不好...
AI: <response>
  <content>发生什么事了吗？我在这里倾听。</content>
  <state_updates>
    <update id="trust" delta="+2" />
  </state_updates>
  <event>
    <importance>6</importance>
    <description>用户主动寻求情感支持</description>
  </event>
</response>
```

### 结果
- 用户看到：*"发生什么事了吗？我在这里倾听。"*
- 信任度：60 → 62
- 新增事件：*"用户主动寻求情感支持"* (重要性: 6/10)

## 🎨 UI功能

### 状态卡片
- 显示状态名称和当前值
- 可视化进度条（颜色可自定义）
- 悬停显示编辑/删除按钮

### 事件列表
- 按重要性排序
- 显示时间戳
- 星级表示重要性（最多5颗星）

### 实时更新
- 对话后状态值自动变化
- 新事件自动添加到列表顶部
- 支持随时编辑状态配置

## 💡 使用技巧

### 1. 合理设置状态
- 不要设置过多状态（建议3-5个）
- 选择有意义的状态名称
- 设置合理的数值范围

### 2. 事件记忆
- AI会自动评估事件重要性
- 重要性高的事件会更影响后续对话
- 系统会保留最近20个事件供AI参考

### 3. 调试
查看浏览器控制台可以看到：
```
[State] 状态更新: [{id: "affection", delta: 5}]
[Event] 新事件: {importance: 8, content: "..."}
```

## 🔍 高级功能

### 自定义状态语义
通过描述字段，可以给AI提供更多上下文：
```
名称: 好感度
描述: 角色对用户的情感程度，影响对话的亲密度和关心程度
```

### 事件重要性级别
- **1-3**: 日常对话
- **4-6**: 有意义的互动
- **7-8**: 重要的情感交流
- **9-10**: 关键性事件

## 📝 注意事项

1. **兼容性**: 只支持使用XML格式的LLM模型
2. **性能**: 状态和事件会增加prompt长度，建议合理控制数量
3. **隐私**: 所有数据存储在本地浏览器IndexedDB中
4. **重置**: 切换对话时，状态和事件不会共享
